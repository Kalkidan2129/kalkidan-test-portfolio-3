
# Revenue Forecasting and Performance Analysis of Vodafone Qatar (2021–2023)

## Project Summary

This project transforms Vodafone Qatar’s 2021‑2023 telecom data into a unified, ML‑ready dataset using NexusMax, Python, and Microsoft Fabric. The resulting analytics platform delivers a Power BI dashboard that visualizes revenue, churn, ARPU, and network performance, enabling finance and strategy teams to forecast quarterly revenue and identify churn risks.

---

## Business Problem

Vodafone Qatar’s finance and strategy teams lacked a consolidated view of revenue, churn, and network performance across regions and segments, making accurate forecasting and budget allocation difficult. Quarterly revenue was rising, yet churn and subscriber behavior fluctuations introduced uncertainty in planning.

---

## Objective

- Consolidate and clean raw telecom data into a single, time‑aligned dataset.
- Develop regression models to forecast quarterly revenue with higher accuracy.
- Create interactive Power BI dashboards to track key KPIs and highlight churn risks.

---

---

## Project Workflow

- 1. Extract and clean raw Vodafone data, standardizing metrics and handling missing values.
- 2. Build a centralized Date table and engineer features such as ARPU, churn, and network availability.
- 3. Train supervised regression models to forecast quarterly revenue.
- 4. Deploy models and export results to Microsoft Fabric for storage.
- 5. Build Power BI dashboards to visualize revenue, churn, and network KPIs and publish to Colaberry Power BI Cloud.

---

## Key Insights

- Total revenue grew 14% QoQ to $108K, driven largely by Doha’s 72M subscribers.
- High‑value prepaid churn exceeded 2.5%, rising 0.3% QoQ, threatening revenue stability.
- Network availability remained 99.95%, supporting strong customer experience and retention.
- Churn in Mobile Postpaid and Fixed Broadband (2.41%–2.74%) was higher than the overall rate, indicating potential $8–12K annual revenue loss.
